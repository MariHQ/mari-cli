# Asset: Code of Conduct

The community's behavioral contract. Convention: the [Contributor Covenant](https://www.contributor-covenant.org)
v2.1, which most projects adopt verbatim or lightly adapt. Structure check:
`mari asset check <file>`. Scaffold: `mari asset scaffold code-of-conduct "<project>"`.

## Canonical structure (required)
- **Our Pledge** — the community's commitment, first person plural.
- **Our Standards** — examples of expected and unacceptable behavior.
- **Enforcement** — how to report, and how reports are handled.

Recommended: **Scope** (where it applies), **Attribution** (keep it if you adapt the Covenant).

## Tone
- Keep the pledge in "We pledge…" voice; it's a shared commitment, not a rule handed down.
- The reporting section must name a **real** contact channel — never a placeholder.
- If you adapt the Contributor Covenant, preserve the Attribution and version.

## Review rubric
1. All required sections present.
2. Enforcement gives a working, private reporting contact.
3. Standards list both encouraged and unacceptable behavior concretely.
4. Attribution/version retained if adapted from a known template.
5. No placeholder emails or "[INSERT CONTACT]" left in.

Leans on: structure check + `assistant-meta` (placeholder detection), `vague-link-text`.
