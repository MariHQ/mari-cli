# critique — editorial judgment, not mechanics

Assess the writing the way an editor would, sitting on top of the mechanical findings.

## Flow
1. Run the detector first so the prose-level critique stands on concrete findings.
2. Judge four dimensions; score each 1–5 and justify:
   - **Argument & structure** — does it lead somewhere; is the order right; is the AI template
     (intro / 3 sections / restated conclusion) avoided?
   - **Clarity** — can the intended reader follow it on first pass?
   - **Voice fidelity** — does it match PRODUCT.md's voice and avoid the anti-references?
   - **Reader experience** — pace, rhythm, payoff; is anything boring or bloated?
3. Produce a **scored snapshot** (the four scores + the top 3–5 concrete issues). Save it so
   `polish` can consume it as its backlog.
4. Recommend the next command.

Critique names problems and gives direction; it doesn't rewrite. That's `polish`/`deslop`/`tighten`.
