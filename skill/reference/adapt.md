# adapt — adapt a piece for a different channel

Same substance, different channel: docs → release note, long-form → social, prose → UI
microcopy, formal → email. Re-pick the register and rebuild for the new medium.

## Flow
1. Identify the source and target channels, and load the target's register reference.
2. Re-pick the register: length, tone, structure, and formatting all change with the channel.
   A release note is not a trimmed doc; a tweet is not a shortened paragraph.
3. Rewrite for the target — keep the load-bearing facts, drop what the new channel doesn't
   carry, add what it needs (a hook for social, a CTA for email, a heading for docs).
4. Run the detector under the target register and clean up.

## Guardrails
- Adapt, don't truncate. Each channel has its own shape.
- Keep facts consistent across versions; if numbers or names appear, they must match the source.

Leans on: the target register reference + `wordy-phrase`, `terminology-consistency`.
