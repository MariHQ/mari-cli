# draft — outline, then write a piece end-to-end in the project voice

Build → write → self-edit. Use when the user wants a new piece, not an edit of existing text.

## Flow
1. If no structure exists, run `outline` internally first — agree the shape before drafting.
2. Draft the whole piece in the project's voice (PRODUCT.md voice + register reference). Write
   like the representative file you sampled, not a generic template.
3. Self-run `deslop` then `tighten` on your own draft before presenting it — the first draft is
   not the deliverable.
4. Run the detector over the result; clear what it flags. Present the piece plus a one-line note
   on the structure you chose.

## Guardrails
- Don't ship the uniform "intro / three sections / conclusion" template — vary the shape to the
  argument (see `outline`).
- Facts you assert must trace to something real; if you invent a number or name, flag it for the
  user rather than presenting it as fact.

Leans on: the full registry (it's a fresh draft, so everything applies) + `conclusion-restate`.
