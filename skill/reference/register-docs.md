# Register: docs (technical documentation, API docs, READMEs, guides)

The default register. Optimize for a reader trying to *do* something.

## Bar
- **Voice:** direct, second person, present tense, imperative for steps ("Run the migration").
- **Sentence ceiling:** ~25 words. Prefer one idea per sentence.
- **Structure:** task-oriented headings (sentence case), short paragraphs, lists only for real enumerations or steps — not as a substitute for prose.
- **Numbers/terms:** consistent terminology (glossary-driven); spell out single digits; define
  acronyms on first use.
- **Tone:** confident and plain. No marketing, no hype, no "easy/simply" (it isn't, for everyone).

## Detector emphasis
Strict on: `long-sentence`, `passive-voice`, `undefined-acronym`, `terminology-consistency`,
`vague-link-text`, `minimizing-words`. Relaxed on: editorial flourish (there shouldn't be any).
