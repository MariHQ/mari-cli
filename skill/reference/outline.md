# outline — plan structure and argument before writing a word

Output an annotated outline, not prose. The point is to settle the shape before committing
sentences to it.

## Flow
1. Read the topic and the register reference. Identify the reader's goal and what they must
   walk away knowing.
2. Produce a structured outline: each section gets a one-line purpose and the key claim or beat
   it carries. Note where examples, evidence, or transitions go.
3. Vary the section cadence to the argument — resist the reflexive "intro / three sections /
   conclusion" template.
4. Hand back the outline for sign-off. Don't draft prose yet (that's `draft`).

## Guardrails
- An outline is a plan, not a summary — every line should constrain the eventual draft.
- Flag any section whose purpose you can't state in one line; that's usually a section that
  shouldn't exist.

Leans on: `conclusion-restate` (don't end by restating the open), `section-parallelism`.
